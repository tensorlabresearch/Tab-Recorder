Local recording, transcription, and storage.

Recordings Folder

Recordings are saved to your browser's Downloads folder under Tab Recorder/.

Local Transcription

Transcripts are produced locally with Transformers.js (whisper via onnxruntime-web). When available, WebGPU is used for a significant speedup; otherwise the WASM CPU backend runs.

Browser AI

When Chrome's built-in Gemini Nano model is already available on this device, Tab Recorder can produce a one-sentence description and a short summary for each recording from its transcript. Inference runs locally; no audio or text leaves your machine.

Status Detecting...

When enabled, a summary file is generated automatically after each transcription completes. Disabled until Browser AI is detected.

Tab Recorder never triggers the model download itself. To enable Nano, toggle chrome://flags/#prompt-api-for-gemini-nano and chrome://flags/#optimization-guide-on-device-model, then visit chrome://components and update "Optimization Guide On Device Model".

Speaker Detection

Optional speaker diarization, also fully local. Uses WavLM (Microsoft, Apache-2.0) through Transformers.js to produce a speaker embedding per utterance, then clusters embeddings to label "Speaker 1", "Speaker 2", etc. in the transcript. The model is downloaded on demand and cached by the browser.

About

MP3 conversion uses LAME via lamejs (LGPL-3.0). Speech recognition uses Transformers.js (Apache-2.0) running ONNX Whisper models.
